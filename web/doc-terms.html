RDF graph terms are represented by the type. These are RDF terms: IRI, literal or blank node. In the graph, the predicate of a triple can only be a IRI.

All strings are considered to be UTF8-encoded. The module provides some functions to manipulate UTF8 strings.

The module defines functions to create and print RDF terms. For example, creates a literal RDF term from an integer, setting the IRI of the datatype to <xsd:integer>:

let int_lit = Rdf_term.mk_literal_int 42;; let term42 = Rdf_term.Literal int_lit;;

Let's get a string representation of the term:

print_endline (Rdf_term.string_of_term term42) ;;

The module defines the type to represent IRIs and contains functions to create IRIs and perform other manipulations:

let iri_s = "http://foo.bar.net/";; let iri = Rdf_iri.iri iri_s;; let iri2 = Rdf_iri.set_fragment iri (Some "id42");; Rdf_iri.compare iri iri2;; Rdf_iri.string iri2;;

The module defines some commonly used IRIs: datatypes, classes, property names, ...:

Rdf_iri.string (Rdf_rdf.xsd_integer);; Rdf_iri.string (Rdf_rdf.xsd_double);; Rdf_iri.string (Rdf_rdf.rdf_about);;
