OCaml-RDF 0.7.0
September 24, 2013

Release 0.7.0 of OCaml-RDF is available from the download page. It contains a lot of changes:

  • Rdf_iri now replaces Rdf_uri,
  • Rdf_sparql.parse_from_* renamed to query_from_*,
  • Rdf_uri.Invalid_url renamed to Invalid_uri,
  • Rdf_ttl.from_* do not return a graph, as the given one is modified in place,
  • The base argument of Rdf_ttl.from_* and Rdf_xml.from_* is now optional; if not given, the graph name is used as base,
  • Add namespace modules: Rdf_foaf , Rdf_earl , Rdf_foaf , Rdf_foaf , Rdf_rdfs ,
  • Partial fix: implement simple version of Sparql's builtin funs LCASE and UCASE,
  • Upgrade turtle parser/printer to W3C RC of 2013/02/19, passes all tests of the Turtle Test Suite except the test containing an invalid UTF-8 character,
  • Do not store namespaces in the graph itself (that was a very bad idea); add functions to add and remove namespaces for a graph,
  • Functor to create new backends now allows to keep a backend-specific representation of terms during Basic Graph Pattern evaluation; this is used to optimize database backends,
  • New "mysql2" backend, faster for queries but slower for insertions,
  • MySQL backend now supports the "engine" option, default is INNODB, but it can be "MyISAM" (faster, but does not support transactions),
  • Remove graph debug dump which was performed at each sparql query evaluation,
  • Rdf_node renamed to Rdf_term ,
  • Rdf_node.node renamed to Rdf_term.term ,
  • Rdf_node.NSet renamed to Rdf_term.TSet ,
  • In triples, predicates are now IRIs and not any term.
  • + lots of bug fixes.
